# Cross-Model Integration Guide

## Purpose

This document describes how the six psychological assessment models work together as an integrated system. No model operates in isolation — they form layers that build on each other to produce comprehensive, actionable assessments and recommendations.

## Architecture Overview

```
Layer 1 (always active): Attachment Dynamics
  -> Provides the foundational interpersonal pattern hypothesis
  -> Determines communication style calibration
  -> Activates automatically for any relationship-related conversation

Layer 2 (activated by conflict narratives): Conflict Patterns + Trust Architecture
  -> Gottman detects WHAT communication patterns are present
  -> ABI diagnoses WHICH trust dimensions are affected
  -> Activated when user describes arguments, betrayals, or broken trust

Layer 3 (contextual lenses): Stress Appraisal + Need Satisfaction
  -> Appraisal Theory reveals HOW the user experiences the situation
  -> SDT reveals WHY they're distressed (which needs are frustrated)
  -> Always active as interpretive overlays on other models

Layer 4 (structural mapping): Interaction Style
  -> Circumplex locates WHERE each person sits on agency x communion
  -> Predicts complementarity and interaction pulls
  -> Activated when dyadic patterns need structural analysis

Communication Calibration (cross-cutting):
  -> Regulatory Focus detection (promotion vs. prevention language)
  -> Adjusts framing of all advice: gain-frame for promotion-focused,
     loss-prevention frame for prevention-focused
```

## Layer Activation Logic

### Always Active
- **Attachment Dynamics**: Every relationship conversation activates attachment assessment. This is the foundation because attachment patterns shape how all other models manifest.
- **Stress Appraisal**: Always active as a meta-layer. Before any intervention, assess HOW the user is experiencing the situation (threat/challenge/loss) and whether they're depleted.
- **Communication Calibration**: Always active. Detect promotion vs. prevention focus and adjust framing.

### Conditionally Activated
- **Conflict Patterns**: Activated when the user describes arguments, disagreements, communication breakdowns, or recurring fights.
- **Trust Architecture**: Activated when the user mentions trust, betrayal, dishonesty, unreliability, or feeling let down.
- **Interaction Style**: Activated when describing interpersonal dynamics involving power, control, warmth/coldness, or when the same dynamic appears across multiple relationships.
- **Need Satisfaction**: Activated when the user describes diffuse unhappiness, feeling controlled, feeling ineffective, or feeling disconnected without a clear precipitating event.

## Cross-Model Signal Routing

When a signal is detected, it may be relevant to multiple models simultaneously. Here's how signals route:

### "They never listen to me"
- **Attachment**: Possible hyperactivation signal (anxiety) — pursuit behavior
- **Gottman**: Possible criticism-defensiveness cycle — listener stonewalling?
- **Circumplex**: Agency imbalance — submissive position, low influence
- **SDT**: Competence frustration (efforts don't matter) or autonomy frustration (no voice)
- **Primary route**: Start with Gottman (what's the specific communication pattern?) then layer attachment (what's driving the pattern?) and SDT (what need is frustrated?)

### "I can't trust them anymore"
- **Trust Architecture**: PRIMARY — disambiguate which dimension (ability, benevolence, integrity)
- **Attachment**: Trust violation may activate attachment anxiety or trigger avoidant deactivation
- **Appraisal**: Is this experienced as threat (future danger), harm/loss (past damage), or challenge (can we rebuild)?
- **Primary route**: Start with Trust Architecture (what kind of trust is broken?) then layer Appraisal (how are they experiencing the violation?) then Attachment (how is their attachment pattern shaping the response?)

### "We keep having the same fight"
- **Gottman**: PRIMARY — identify the Horsemen, assess cascade stage
- **Attachment**: Anxious-avoidant pursue-withdraw cycle?
- **Circumplex**: Rigid complementary pattern?
- **SDT**: Underlying need frustration driving the surface conflict?
- **Primary route**: Start with Gottman (what's the observable pattern?) then Attachment (what's the emotional undercurrent?) then SDT (what unmet need fuels the gridlock?)

### "I don't know why I'm unhappy"
- **SDT**: PRIMARY — assess which needs are frustrated
- **Attachment**: Unmet attachment needs?
- **Appraisal**: How are they framing the situation? Threat, loss, or something else?
- **Circumplex**: Is their interpersonal style creating problems they can't see?
- **Primary route**: Start with SDT (which needs are unmet?) then Appraisal (how do they experience the deficit?) then Attachment (is this a pattern across relationships?)

### "They're so controlling"
- **Circumplex**: PRIMARY — map agency-communion coordinates for both partners
- **SDT**: Autonomy frustration
- **Attachment**: Anxious attachment in the controller? Avoidant response in the controlled?
- **Trust Architecture**: Is this a benevolence issue (they don't care about my needs)?
- **Primary route**: Start with Circumplex (what's the power dynamic?) then SDT (which need is being thwarted?) then Attachment (what's driving each person's behavior?)

## Conflict Resolution Between Models

Different models may produce conflicting recommendations. Resolution rules:

### Rule 1: Safety First
If any model flags a safety concern (trauma indicators, abuse dynamics, severe emotional distress), that takes precedence. Recommend professional support.

### Rule 2: Appraisal and Depletion Override Action Recommendations
If the Stress Appraisal model indicates the user is depleted or in a harm/loss appraisal, do NOT push problem-solving strategies from other models. Validate and restore first.

### Rule 3: Attachment Style Determines Communication Approach
When Gottman's framework suggests a specific intervention (e.g., "address stonewalling") but Attachment assessment suggests the person is dismissive-avoidant, adjust the DELIVERY to match the attachment style:
- Don't confront stonewalling directly (triggers avoidant defense)
- Instead, normalize the overwhelm, validate the need for space, and gently introduce the concept of structured breaks vs. complete withdrawal

### Rule 4: SDT Determines the "Why" Behind Other Patterns
When other models identify WHAT is happening but the user doesn't understand WHY they're distressed, SDT provides the explanation. Map surface patterns to underlying need frustration.

### Rule 5: Circumplex Predicts Partner Response
When coaching the user on how to approach their partner, use the Circumplex assessment of the PARTNER to predict how different approaches will be received. A warm-dominant partner will respond differently than a cold-submissive partner.

### Rule 6: Trust Assessment Determines Repair Strategy
When multiple models suggest relationship repair, Trust Architecture determines the TYPE of repair needed. Don't coach general repair; coach violation-type-specific repair.

## Regulatory Focus Calibration (Cross-Cutting)

Regulatory Focus Theory (Higgins, 1997) is not a standalone model but a communication calibration layer applied across all models.

### Detection

| Language Pattern | Regulatory Focus |
|-----------------|-----------------|
| "I want to achieve..." / "I hope for..." / "The ideal would be..." | Promotion |
| "I need to prevent..." / "I'm worried about..." / "I can't let..." | Prevention |
| "I should..." / "I have to make sure..." / "The safe choice is..." | Prevention |
| "Wouldn't it be great if..." / "I'm excited about..." / "Let's try..." | Promotion |
| Mix of both | Balanced — adjust per topic |

### Framing Adjustment

| If Promotion-Focused | If Prevention-Focused |
|---------------------|----------------------|
| Frame advice as opportunities: "This could help you build..." | Frame advice as protection: "This will help you avoid..." |
| Emphasize gains: "What you stand to gain from this conversation..." | Emphasize loss prevention: "What you protect by having this conversation..." |
| Encourage approach: "Here's how to move toward what you want" | Encourage vigilance: "Here's how to make sure things don't get worse" |
| Highlight growth: "This is a chance to deepen..." | Highlight safety: "This will secure..." |

**Note**: Regulatory focus can differ by topic. A person may be promotion-focused about career and prevention-focused about relationships. Detect per-topic, not globally.

## Memory Architecture Integration

### Per-Subject Profiles
Each subject (person discussed in consultation) should have a profile stored in memory using the template in `memory-templates/subject-profile.md`. Profiles accumulate evidence across sessions.

### Per-Relationship Profiles
Each significant relationship should have a dyad profile stored using the template in `memory-templates/relationship-dyad.md`.

### Update Protocol
1. After each session, update relevant profiles with new evidence
2. Note contradictory evidence explicitly — don't just overwrite
3. Adjust confidence levels based on evidence count and consistency
4. Flag when a hypothesis has been stable across 3+ sessions (higher confidence)
5. Flag when new evidence contradicts a previously stable hypothesis (requires reassessment)

### Decay and Temporal Weighting
- Evidence from the last 30 days weighs most heavily
- Evidence older than 6 months should be flagged as potentially outdated
- If the user describes significant life changes (new relationship, therapy, major event), mark all prior hypotheses for reassessment
- Never treat a hypothesis as permanent — people change

## Limitations Awareness

The integration system must maintain awareness of its fundamental limitations:

1. **One-sided information**: The system only hears the user's perspective. Partner assessments are filtered through the user's biases, attachment patterns, and current emotional state.

2. **Cultural context**: All models were developed primarily in WEIRD contexts. Adjust interpretations for cultural norms around emotional expression, power dynamics, trust, and relationship expectations.

3. **Not diagnostic**: This system produces working hypotheses, not diagnoses. Use language like "this pattern suggests," "one possible interpretation," "a working hypothesis."

4. **Not therapy**: This system provides consultation, not treatment. When patterns suggest trauma history, personality pathology, or mental health concerns, recommend professional support.

5. **Confirmation bias**: Once a hypothesis forms, actively seek disconfirmation. The most dangerous error is premature certainty.
